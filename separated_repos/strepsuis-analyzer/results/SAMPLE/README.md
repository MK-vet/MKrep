# Sample E2E Results Structure

This directory demonstrates the output structure generated by the `strepsuis-analyzer-e2e` command.

## Directory Layout

When you run the E2E analysis, a timestamped directory is created:

```
results/strepsuis-analyzer-YYYYMMDD-HHMM/
├── example/                      # Results from example dataset
│   ├── validation/
│   │   ├── validation_report.csv
│   │   └── summary.log
│   ├── stats/
│   │   ├── correlation_matrix.csv
│   │   ├── correlation_heatmap.png
│   │   ├── normality_results.csv
│   │   ├── qq_plots.png
│   │   ├── hypothesis_tests.csv
│   │   ├── corrected_pvalues.csv
│   │   └── meta_analysis_results.csv
│   ├── visualizations/
│   │   ├── histogram.png
│   │   ├── scatter_plot.png
│   │   ├── box_plot.png
│   │   └── violin_plot.png
│   ├── clustering/
│   │   ├── kmeans_labels.csv
│   │   ├── elbow_plot.png
│   │   ├── kmodes_labels.csv
│   │   ├── hierarchical_labels.csv
│   │   ├── dendrogram.png

├── synthetic/                    # Results from synthetic dataset
│   └── (same structure as above)
└── logs/
    └── run.log                   # Detailed execution log
```

## File Descriptions

### Validation (`validation/`)
- **validation_report.csv**: Summary of data validation checks
- **summary.log**: Text summary of validation results

### Statistics (`stats/`)
- **correlation_matrix.csv**: Correlation coefficients between variables
- **correlation_heatmap.png**: Visual representation of correlations
- **normality_results.csv**: Shapiro-Wilk normality test results
- **qq_plots.png**: Q-Q plots for normality assessment
- **hypothesis_tests.csv**: Results from t-tests, ANOVA, etc.
- **corrected_pvalues.csv**: Bonferroni and FDR corrected p-values
- **meta_analysis_results.csv**: Fixed and random effects meta-analysis

### Visualizations (`visualizations/`)
- **histogram.png**: Distribution plots
- **scatter_plot.png**: Bivariate relationships with regression
- **box_plot.png**: Box plots for group comparisons
- **violin_plot.png**: Distribution with density estimation

### Clustering (`clustering/`)
- **kmeans_labels.csv**: K-Means cluster assignments
- **elbow_plot.png**: Elbow plot for optimal K selection
- **kmodes_labels.csv**: K-Modes cluster assignments (for categorical/binary data)
- **hierarchical_labels.csv**: Hierarchical clustering assignments
- **dendrogram.png**: Hierarchical clustering dendrogram
- **dbscan_labels.csv**: DBSCAN cluster assignments

### Phylogenetics (`phylogenetics/`)
- **tree_plot.png**: Phylogenetic tree visualization
- **rf_distances.csv**: Robinson-Foulds distances between trees
- **phylo_diversity.csv**: Faith's Phylogenetic Diversity metrics
- **cophenetic_correlation.csv**: Tree quality assessment

### ETL (`etl/`)
- **pivot_table.csv**: Reshaped data in pivot format
- **aggregated_data.csv**: Group-by aggregations
- **normalized_data.csv**: Z-score or min-max normalized data
- **merged_data.csv**: Joined datasets

### Reports (`reports/`)
- **final_report.xlsx**: Multi-sheet Excel workbook with all analyses
- **final_report.html**: Self-contained HTML report with visualizations

### Logs (`logs/`)
- **run.log**: Detailed execution log with timestamps, operations, and file paths

## Usage

To generate these results, run:

```bash
# Install the package
pip install -e .

# Run E2E analysis on both datasets
strepsuis-analyzer-e2e --dataset both

# Or run on specific dataset
strepsuis-analyzer-e2e --dataset example
strepsuis-analyzer-e2e --dataset synthetic

# Customize results directory
strepsuis-analyzer-e2e --dataset both --results-dir custom_results/
```

## CI Artifacts

When run in GitHub Actions CI, the complete results directory is uploaded as an artifact:
- Artifact name: `e2e-results-{python-version}`
- Retention: 7 days
- Location: Actions tab → Workflow run → Artifacts section

## Notes

- Results directories are timestamped to prevent overwrites
- Large binary files (PNG, XLSX, HTML) are excluded from git commits
- For git commits, only `.gitkeep` files and this README are tracked
- Full results are available as CI artifacts
- Typical total size: 1-3 MB per dataset
